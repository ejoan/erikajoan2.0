Chango Dashboard: Redesign
The main aim of this project was to improve the navigation and content hierarchy by consolidating the top and left navigation. Responsibilities included research, preliminary sketches, mock-up phases, user testing and front end development. The Result — one top navigation with a breadcrumb for context clarity, easy back tracking, and a face-lift on key features of the dashboard.
1. Navigation
Left navigation has been integrated into a horizontal top header to create more retail space for graphs and reporting features across the dashboard.

  • Before
2. Datepicker
The size of the datepicker has been made smaller to make room for the links now included in the top navigation and includes the exact date range.
  • Outcome
  • Before
3. Tickets

The dashboard ticketing system is used to create tickets for tasks or issues that need to be addressed by specialty users such as QA, Optimization, or Data Analytics. The re-skin of the tickets component aimed to clarify who the reporters and the assignees of the tickets were, as well as establish the originating context of the tickets.

4. Notes

The notes component proved to be a very useful tool for ad operations users to communicate with one another about any updates on any accounts, campaigns or tactics. This design revamp introduced pinning notes, and contextual notes.

Note: The outcome image has been captured after Chango was acquired by The Rubicon Project.

  • Before
5. Activity Logs

The activity logs help users track all the changes that have been made to an account or any object pertaining to it. These logs provide the details of who performed the changes, the old value and the new value of the changed item in a column format.

Note: The outcome image has been captured after Chango was acquired by The Rubicon Project.

6. Footer
A footer was introduced when it was decided that we were going to get rid of the left navigation. Account meta data normally found at the bottom of the navigation on the left hand side was moved into a page footer for easy reference. More account information was added into the footer as the dashboard and its users evolved.
7. Report Builder
This is one of the many interfaces that benefit greatly from more horizontal space, having retired the left navigation. This area was used to build custom reports that were exportable and available to send as email subscriptions.
